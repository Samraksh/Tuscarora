117  /*
118  * @brief Request a particular type of schedule from the waveform
119  *
120  * <b>Semantic Behavior:</b> This method enables the waveform to share its schedule information with the framework. This is an optional method. If the
121  * waveform has no schedule or if the waveform does not want to share this information with the Framework, this method can be ignored. The parameters
122  * specify the RequestId, the neighbor for which the schedule information is requested and the type of the schedule.
123  *
124  * The waveform should regenerate a response to this request through the Control Response Event, with the same RequestId. The status of that response
125  * message should be true. A WF_ScheduleUpdateParam structure with the schedule information should be created and copied into the data part of the response
126  * and the size of the data should be set as the size of WF_ScheduleUpdateParam object being sent to the Framework.
127  *
128  * Since this is an optional feature for waveforms, if it choose to implement it, the waveform should set the schedulerInfoSupport field in its Attribute
129  * as true whenever it responds to the AttributeRequest.
130 
131  *
132  * @param nodeid Specifies the neighbor for which the schedule is requested
133  * @param type Enum that specifies the type of schedule. There are 3 types of schedules, sending, receiving and link estimation.
134  * @return void.
135  *
136  */
137  virtual void ScheduleRequest (RequestId_t rId, NodeId_t nodeid, WF_ScheduleTypeE type) {}
138 };
